## 📊 Executive Summary

모두콘 2025 디지털 컨퍼런스 북은 **게임화된 퀘스트 기반 컨퍼런스 경험**을 제공하는 Progressive Web App입니다. 현재 기본 인프라와 부스/포스터 기능은 완료되었으나, **QR 스캐너 UX 개선**과 **실제 세션 데이터 통합** 작업이 필요한 상태입니다.

## 🎯 1. 재미 (Fun) - 7/10점

### 강점 ✅
1. **퀘스트 시스템의 게임화**: 컨퍼런스를 RPG 게임처럼 탐험하는 독특한 접근
2. **히든 퀘스트**: 숨겨진 요소 발견의 재미 제공
3. **진행도 추적**: 프로그레스 바, 체크리스트로 성취감 부여
4. **배지 시스템**: 활동 보상으로 동기부여

### 약점 ⚠️
1. **QR 스캐너 UX 미흡**:
   - 현재 구현: 기본적인 html5-qrcode 라이브러리만 사용
   - 문제점: UI가 다른 요소와 분리되지 않음, 사용성 낮음
   - 개선 필요: 원형 UI, 예시 QR 아이콘, 명확한 가이드 메시지

2. **실제 세션 데이터 부재**:
   - Google Sheets에 33개 세션 데이터 존재
   - 프론트엔드에는 하드코딩된 더미 데이터만 표시
   - 실제 연사, 시간, 트랙 정보와 연동 안 됨

3. **퀘스트 자동 생성 알고리즘 미구현**:
   - PRD에 명시된 개인화 추천 시스템 없음
   - 관심사 기반 동선 최적화 미구현

### 개선 방향 💡
- QR 스캐너를 중앙 원형 버튼으로 재설계
- 세션 데이터 Google Sheets 연동 완료
- 퀘스트 추천 로직 Phase 1 버전 구현

**채점 근거**: 기획은 8점급이나 핵심 재미 요소인 QR 스캔 UX와 데이터 연동이 미완성 → **7/10점**

   - 모두콘: 퀘스트 기반 탐험 + 개인화 동선 + 보상 시스템

2. **오픈소스 보상 전략**:
   > "상업적 굿즈 대신, LAB의 순수 AI 데모를 퀘스트로 경험하고, 앱 소스코드를 보상으로 받아가는 유일한 컨퍼런스"
   - 기술 중심 커뮤니티에 최적화된 차별화 전략

4. **데이터 기반 의사결정 구조**:
   - 실시간 혼잡도 표시로 참가자 경험 개선
   - 체크인/체크아웃 데이터로 행사 운영 최적화
   - 다음 행사 기획에 활용 가능한 인사이트 축적

### 개선 가능 영역 🔧

**채점 근거**: 컨퍼런스 앱의 새로운 패러다임 제시, 실행 가능성 높은 혁신 → **9/10점**

## 📋 신규 요구사항 분석

### 요구사항 1: QR 스캐너 UI 개선

#### 현재 상태
```typescript
// src/components/qr/QRScanner.tsx
<div id="qr-reader" className="w-full max-w-md" />
```
- 기본 html5-qrcode UI
- 다른 요소와 통합되지 않음
- 사용자 가이드 부족

#### 요구사항
> "정 가운데 다른 UI와 분리하여 원형 안에 예시 QR 이미지가 있는 아이콘으로 QR을 찍는 기능"

#### 평가: 9/10점 (매우 중요, 명확함)
- **중요도**: 🔴 Critical - 핵심 사용자 경험
- **명확성**: ✅ 구체적 (원형, 중앙 배치, 예시 이미지)
- **실현 가능성**: ✅ 기술적으로 간단
- **예상 작업**: 2-4시간
  - 원형 버튼 컴포넌트 제작
  - QR 아이콘 SVG 추가
  - 카메라 모달 재설계
  - 후방 카메라 자동 선택 검증

### 요구사항 2: 세션 실제 데이터 통합

#### 현재 상태
- Google Sheets: 33개 세션 × 14개 필드
- 프론트엔드: 하드코딩 더미 데이터
- 백엔드: Google Sheets API 준비 완료

#### 요구사항
> "세션이 실제 정보로 대체되어야합니다. Google Sheets '세션' 시트 참고"

#### 평가: 10/10점 (필수, 명확함)
- **중요도**: 🔴 Critical - 행사 필수 정보
- **명확성**: ✅ 데이터 소스 명시
- **실현 가능성**: ✅ 인프라 준비됨
- **예상 작업**: 4-6시간
  - Google Sheets API 연동 함수 작성
  - Session 타입 정의 (14개 필드)
  - 트랙별 필터링 로직
  - 시간대별 정렬
  - 캐싱 전략 (5분 갱신)

#### 데이터 필드 매핑
| Sheets 컬럼 | 프론트 필드 | 변환 필요 |
|------------|------------|----------|
| 번호 | id | - |
| 트랙 | track | - |
| 위치 | location | - |
| 발표-시간 | startTime, endTime | 파싱 필요 |
| 연사-명 | speaker | - |
| 발표-제목 | title | - |
| 발표-내용 | description | - |
| 키워드1-3 | tags | 배열 변환 |

## 🎨 UX/UI 평가

### 디자인 시스템
- **일관성**: 6/10 - 컴포넌트별 스타일 통일성 부족
- **접근성**: 5/10 - ARIA 라벨, 키보드 네비게이션 미흡
- **반응형**: 7/10 - 모바일 최적화는 양호
- **성능**: 7/10 - 초기 로딩은 빠르나 최적화 여지

### 사용자 흐름
```
현장 도착
  ↓
QR 코드 스캔 (입구/데스크)
  ↓
로그인 (이름 + 전화번호)
  ↓
디지털 서명 ✅
  ↓
관심사 선택
  ↓
퀘스트 맵 생성 ⚠️ (미구현)
  ↓
세션/부스/포스터 탐험
  ↓
QR 인증 ⚠️ (UX 개선 필요)
  ↓
배지 획득 ⚠️ (미구현)
```

### 권장 개선
```typescript
// 캐싱 전략
const CACHE_DURATION = 5 * 60 * 1000; // 5분

// SWR 패턴 적용
export async function fetchSessions() {
  const cached = sessionStorage.getItem('sessions');
  const timestamp = sessionStorage.getItem('sessions_timestamp');

  if (cached && timestamp) {
    const age = Date.now() - parseInt(timestamp);
    if (age < CACHE_DURATION) {
      return JSON.parse(cached);
    }
  }

  // Google Sheets API 호출
  const data = await fetchFromGoogleSheets();

  sessionStorage.setItem('sessions', JSON.stringify(data));
  sessionStorage.setItem('sessions_timestamp', Date.now().toString());

  return data;
}
```
