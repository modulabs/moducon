## 🏗️ 프로젝트 아키텍처

### Frontend (GitHub Pages)
- **Framework**: Next.js 14+ (Static Export Mode)
- **UI**: React 18+ + Tailwind CSS + shadcn/ui
- **State**: Zustand 또는 React Context
- **PWA**: Service Worker + Offline Mode
- **Deployment**: GitHub Pages (CDN)
- **Domain**: moducon.vibemakers.kr (권장)

### Backend (기존 서버)
- **API**: REST API (기존 백엔드 서버)
- **Database**: PostgreSQL
- **Auth**: JWT 기반 인증
- **Realtime**: WebSocket 또는 Server-Sent Events
- **Storage**: 파일 저장소 (서명, 이미지 등)

## 💡 주요 결정 사항

### 아키텍처 결정 (v1.1)
- **Before**: Vercel 배포
- **After**: GitHub Pages (프론트엔드) + 기존 백엔드 서버
- **이유**:
  - GitHub Pages: 무료, 무제한 트래픽, 빠른 CDN
  - 기존 백엔드 활용: 빠른 개발, 안정성 확보

### 도메인 결정 (v1.2)
- **Domain**: moducon.vibemakers.kr
- **장점**:
  - 짧고 기억하기 쉬운 URL (23자 vs 32자)
  - QR 코드 단순화 → 스캔 성공률 향상
  - 전문적인 브랜딩

### 로그인 플로우 결정 (v1.3)
- **Flow**: 현장 QR 스캔 → 로그인 (이름+전화번호) → 서명 → 출입증
- **이유**:
  - 사전 신청자 인증 필수화 (보안 강화)
  - 서명 후 출입증 발급 (법적 동의 증빙)
